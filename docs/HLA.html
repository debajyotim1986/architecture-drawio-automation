High-Level Architecture · MCP · GitHub Copilot

AI Architecture-Diagram Copilot

A local Model-Context-Protocol server that turns a Jira requirement into a deck-quality draw.io diagram. A one-line plain-English instruction routes to the right backend prompt; a policy guardrail swaps out restricted services before anything is drawn. GitHub Copilot in VS Code speaks MCP over stdio to a registry of 13 intent-level tools; a domain adapter renders mxGraph XML; everything is written through a path-sandboxed file store. An offline CLI toolchain normalizes the requirement, fills the connector contract, and gates quality before delivery.

Governance — Restricted Services

policy/unavailable_services.md
service name · aliases ban list + a separate 94-row alternate1–4 reference table — the only source of replacements.
  • Hard mandate — read & checked before the first tool call (create & update)
  • Alternates ranked by the Jira scenario; restricted ones skipped
  • Stops & asks — presents every option and waits for your pick

Cross-Cutting Guarantees

  • Restricted-service guardrail — mandatory pre-build gate; stops to ask which approved service to use before any node is drawn
  • Path sandbox — stores reject any path outside their root
  • Verifier gate — exit-2 blocks delivery of a broken diagram
  • Zero-TBD guard — table refuses to embed if any cell is unfilled
  • Deterministic layout — column gap, row stride, sub-lane deconfliction
  • Pluggable icon catalog — GCP icons today (logo_icon_gcp/)
  • Cross-OS bootstrap — same command on Mac / Linux / Windows
