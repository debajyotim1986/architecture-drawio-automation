A local Python Model Context Protocol server that lets any LLM client (GitHub Copilot, Claude Desktop, Cursor) build deck-quality architecture diagrams from a single prompt — pluggable icon catalogs, intent-level layout tools, automatic routing, and a structured connector-detail table generated alongside every diagram.

The Problem

  • Architecture diagrams are manual, slow, and inconsistent — engineers either skip them or hand-draw whiteboards that drift from the code.
  • Different teams pick different shapes, palettes, and conventions, so every diagram has to be re-learned.
  • Connector details (protocol, auth, SLA, monitoring) live in tribal knowledge instead of on the diagram.

The Solution — Framework Architecture

A four-layer pipeline. Each layer has one job and a clear contract with its neighbour, so any layer can be swapped without rewriting the others.

1. LLM ClientCopilot / Claude / Cursor — speaks MCP
2. MCP Server (stdio JSON-RPC)server.py — dispatches calls
3. Intent-Level Toolstools/* — 13 small, composable verbs
4. drawio XML Adapterparser · builder · shapes · icons
5. Sandboxed Storagefile-system, refuses paths outside the root

Pluggable icon catalog (GCP today, AWS/Azure trivial), deterministic auto-layout (column gap, row stride, sub-lane deconfliction), and a verifier that hard-fails before any diagram is delivered.

Engineering Highlights

  • Intent-level API — LLM says "add_container Ingestion" / "add_edge A→B"; the server handles every pixel.
  • Smart routing — orthogonal edges with row-gap waypoints + sub-lane deconfliction prevent overlap.
  • Auto fan-out — parallel connectors spread across the node perimeter automatically.
  • Busy-node cards — nodes with > 3 edges auto-wrap in a rounded card border.
  • Pluggable icon catalog — drop SVGs in logo_icon_*/, the resolver picks them by service name.
  • Zero-TBD guard — connector table refuses to embed if any cell is unfilled.
  • Self-bootstrapping scripts — re-exec under venv; one command works on Mac / Linux / Windows.

Outcomes

  • From ~2 hours of manual drafting per architecture diagram down to one prompt.
  • Visually consistent — same palette, icon system, and routing across every diagram an org produces.
  • Verifiable — a verify_mcp.py exit-2 means the diagram does not ship.
  • Reusable — swap the icon catalog and the framework draws AWS, Azure, on-prem, or domain-specific diagrams.
  • Documented — 12 Mermaid views + tabular component matrix make onboarding minutes, not days.
